PO-to-SO Processing Pipeline

1
Email Intake
Customers send POs to orders@enproinc.com as email with PDF attachment. The agent polls this inbox via Microsoft Graph API. Ariba and Coupa POs also arrive here, sometimes as cXML attachments.
2
Parse PO
PDF: Azure Document Intelligence extracts PO#, ship-to, line items, quantities, prices from the PDF.
cXML: Native XML parsing for Ariba/Coupa OrderRequest format.
CSV: Direct column mapping for structured imports.
3
Customer Match
Match the ship-to name + zip code to a P21 customer_id using the Customer Crosswalk (4,880 entries). Priority: learned exact match > name+zip exact > fuzzy address > PO# pattern.
4
Item Match
For each line item, match the customer's part number to a P21 item using the Customer-Item Crosswalk (22,675 entries). This table maps what each specific customer calls their parts to EnPro's P21 item IDs.
5
Confidence Score
4-dimension scoring:
Customer (30%) — did we identify who this is?
Ship-to (15%) — do we know this delivery address?
Items (45%) — did every line item match a P21 product?
Duplicate (10%) — is this a new PO or a re-send?
GREEN = auto-approve | YELLOW = Brittany reviews | RED = manual entry needed
6
CISM SO Generation
For green/yellow POs, generate a P21 Order/Quote Import CSV matching the CISM batch schema (orderquoteheader + orderquoteline format). This CSV gets picked up by P21's scheduled import job to create the Sales Order.
7
Learning Loop
Every time a PO is approved (automatically or by Brittany), the system remembers:
Customer: "Ship-to 'Waupaca Foundry' at zip 47586 = P21 customer 208672" — instant match next time.
Items: "Customer 208672 calls part PH0260 = P21 inv_mast_uid 7931" — remembered per customer.
PO Link: "PO# 4500867075 = customer 208672" — catches duplicates, confirms identity.
Day 1: mostly yellow/red (learning). Month 1: mostly green (automated).
