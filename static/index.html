PO-to-SO Processing Pipeline

1
Email Intake
Poll orders@enproinc.com via Microsoft Graph API. Classify source (Ariba/Coupa) from sender domain + cXML headers.
2
Parse PO
Extract header + lines from cXML OrderRequest or PDF via Azure Document Intelligence.
3
Customer Match
Match ship-to name + zip to P21 customer_id using crosswalk. Fuzzy fallback with Jaccard + Levenshtein.
4
Item Match
Match each line's supplier_part_id to P21 inv_mast_uid via customer_part_number crosswalk (100% populated from oe_line).
5
Confidence Score
4-dimension scoring: customer (0.30) + ship-to (0.15) + items (0.45) + dedup (0.10). Green/Yellow/Red.
6
CISM SO Generation
Generate P21 Order/Quote Import CSV (header + lines) matching the CISM batch schema. Green = auto-stage.
7
Learning Loop
Every approved match feeds back into crosswalk CSVs. Customer + item + ship-to mappings grow over time.
